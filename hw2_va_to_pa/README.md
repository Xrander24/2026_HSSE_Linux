# Syscall VA->PA

## Предлагаемое решение
Системный вызов для получения реального адреса памяти по виртуальному + pid процесса

Идейно работает так: получаем данные о процессе -> из них достаем данные об его адресном пространстве -> проходом по уровням Page directory добираемся до реального адреса

Так же написаны юзерспейс программы - тесты

## Как запускать
Добавить файлик с сисколом в ядро, в kernel, там же в Makefile дописать ```obj-y += va_to_pa.o```

Добавить сисколл в таблицу arch/x86/entry/syscalls/syscall_64.tbl
дописать строку ```667 common va_to_pa sys_va_to_pa``` или с любым другим номером из свободных

пересобрать ядро через make, статически скомпилить файлик с тестом например user.cpp, закинуть в initramfs, запустить qemu с указанием нового образа ядра, он должен быть тут arch/x86/boot/bzImage
